  - 先验证可行性，后续再扩展App端

## Key Decisions

- **Web优先而非App**：一人公司，先用最低成本验证产品可行性，Web端有人付费后再扩展App端
- **Next.js + Supabase + DeepSeek + Stripe**：技术栈选择，兼顾开发效率、成本和国内开发友好度
- **Next.js API Routes统一处理后端**：不额外使用Supabase Edge Functions，代码集中、部署统一
- **先验价值后置付费**：3轮诊断→展示计划预览→付费墙，转化率优于中途打断
- **免费版限制"调整权"而非"次数"**：免费用户仅查看计划不可调整，制造强烈付费动机
- **3天无门槛试用**：海外SaaS转化率最高的策略
